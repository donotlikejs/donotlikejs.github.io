环境变量PATH的查看，添加

  • 在自主安装的文件经常会用到shell来启动执行，有时候需要自行把程序添加到$PATH之中,当输入可执行程序后，会搜索这个目录列表。
  • mac中有三个级别对环境变量，分别是 系统变量 全局级变量和用户级变量，我们一般只修改 用户级别的环境变量 。
  • 保存环境变量的文件名为 .bash_profile ,位于 home根目录 下。
  • 查看PATH
echo $PATH
  • 添加path
export $PATH=路径
  • 立即生效
source ~/.bash_profile

路径表示 

./    当前目录(可省略 比如 cd ./demo 写成 cd demo)
../   上一级目录
/     绝对路径(根目录)

cd 改变目录

// 进入root根目录
cd /
// 进入用户根目录
cd ~
cd
//进入工作目录的子目录demo
cd ./demo
cd demo
//进入工作目录先前的目录
cd -

touch（新建文件）

mkdir（新建目录）

// 当前目录新建 demo.txt和index.html
touch demo.txt index.html
// 当前目录新建demo文件夹
mkdir demo

创建软链接（类似window快捷方式）

ln -s 源文件 目标文件

rm 删除文件

​ 常用选项

​ -i -r -v 同复制

​ -f: 忽略不存在的文件

rmdir 删除空目录

rm -rf 删除目录 -r递归删除 f无需确认

ls 显示文件列表

常用选项 -l:长格式显示结果

​ -t:按修改时间来排序

​ -a:显示所有文件 -r:相反顺序显示

​ -h:长格式列出时，不已字节显示文件大小

​ -F: 每个文件加上指示符号

​ -d: 列出指定目录中内容

pwd 显示工作目录路径

mv（移动/重命名文件和目录） 和 cp（拷贝文件和目录）

常用选项 -a:复制所有信息和属性

​ -i:重写前确认

​ -r:递归(复制)

​ -u:仅复制目标目录不存在或新于的文件

​ -v:显示详细命令操作信息

//复制file1文件到file2
cp file1 file2
//重命名为file2
mv file1 file2
//移动file1到根目录下的demo目录
mv file1 /demo
