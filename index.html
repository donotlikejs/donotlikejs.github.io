AJAX原生

  • 异步加载必须先实例化一个XMLHttpRequest函数。
let xhr = new XMLHttpRequest();

  • XMLHttpRequest的响应方法
/*
    readyStateChange 用于监听readyState的变化
    status 响应状态码 1xx 2xx 3xx 4xx 5xx
    statusText 响应状态说明
    readyState    请求／响应过程的当前活动状态
        0
        1
        2    发送，已经调用send方法
        3    接收响应数据
        4    表示完成

    responseText    响应主体返回的文本
    responseXML        响应主体返回的XML

*/
  • GET 和 POST 的一般写法
// get
let xhr=new XMLHttpRequest();
url="http://www.abc.com?a=b&c=d";
xhr.open("get",url,true);
xhr.send(null);
xhr.readyStateChange=function(){
  if(xhr.readyState==4&&xhr.status==200){
    console.log(xhr.responseText);
  }else{
    console.log(`error${xhr.status}`);
  }
};

//post
let xhr=new XMLHttpRequest();
url="http://abc.com";
data="a=b&c=d";
xhr.open("post",url,true);
xhr.setRequestHeader("Content-Type","application/x-www-from-urlencoded");
xhr.send(data);
xhr.readyStateChange=function(){
  if(xhr.readyState==4&&xhr.stauts=200){
    console.log(xhr.responseText);
  }else{
    console.log(`error:${xhr.status}`);

  }
}

修饰符

/*

    i    不区分大小
    g    全局匹配
    m    多行匹配
    u    （es6新增）

*/

元字符 . \b \w\d\s

/*
    .          匹配任意字符
    \b         匹配单词边界
    \w        匹配数字，字母，下划线
    \d        匹配数字
    \s        匹配空格
    [abc]    匹配方括号里任意一个字符
    ^        匹配首部
    $        匹配尾部

    --取反--
    \B \W \D \S [^abc]

*/

量词 * + ? {}

/*

    *        匹配任意个
    +        匹配一到多个
    ?        匹配 0 到 1 个
    {n}        匹配 n 个
    {n,}    匹配至少 n 个
    {n,m}    匹配 n 到 m 个

*/

分枝条件 |

/*

a|b         匹配 a 或者 b 

*/

分组 ()

/*

    (exp)            匹配且分组
    (?<name>exp)    匹配且以name名分组
    (?:exp)            只匹配不捕获不分组
    (?=exp)            匹配exp前面位置
    (?<=exp)        匹配exp后面位置
    (?!exp)            匹配后面不跟着exp的位置
    (?<!exp)        匹配前面不跟着exp的位置
    (?#comment)        注释

 */

懒惰限定符

/*

    *?
    +?
    ??
    {n,m}?
    {n,}?

*/

转义字符 \

/*

    需要转义的字符   * . ? + $ ^ [ ] ( ) { } / \ |

*/

<meta http-equiv="X-UA-Compatible" content="edge">

X-UA-Compatible 是针对IE8版本的一个特殊文件头标记，content=”edge” 通知IE用最高级别的可用模式显示内容

http-equiv 类似于http头部协议，用来回应给浏览器一些信息，以帮助浏览器正确的现实网页内容。
一些平常使用的http-equiv的名值对

//10s后刷新页面
<meta http-equiv="refresh" content="10">
//10s后跳转到bing
<meta http-equiv="refresh" content="10" url="http://bing.com">
