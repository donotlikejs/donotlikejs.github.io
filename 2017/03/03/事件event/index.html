事件：用户或者浏览器自身执行的某种动作

事件处理程序（事件监听器）：响应某个事件的函数，事件处理程序的名字以“on”开头。

DOM0级：将一个函数赋值给一个事件处理程序属性

btn.onclick=function(){
}

DOM1级

DOM2级事件处理程序

/*
*1参数为要处理的事件名
*2参数事件处理程序的函数
*3参数为布尔值，true表示捕获阶段调用2，false表示冒泡阶段调用2
    fn在当前作用于运行
*/
addEventListener("",fn,boolean)
/*
参数同上，只是调用变成删除
*/
removeEventListener()

捕获：从不具体的父元素逐步传到具体的元素

冒泡：具体的元素逐步往上传播到不具体的父元素

DOM事件流（DOM2）:分为事件捕获阶段，处于目标阶段和事件冒泡阶段。（但浏览器都支持捕获和冒泡阶段对目标对象进行操作）

事件处理程序分类

HTML事件处理程序 ：写在html标签里

IE事件处理程序：

// fn会在全局作用域中运行
attachEvent()
detachEvent()

事件对象（event）

触发DOM事件时自动产生事件对象event，该对象包含所有与事件有关的信息。

stopPropagation
preventDefault()   取消默认行为
target                返回最初触发事件的元素
type                返回事件类型

//原点为浏览器内容区域左上角
clientX
clientY

//原点为屏幕左上角
screenX
screenY

//FF特有 原点为浏览器内容区域左上角随scroll变化
pageX
pageY

事件委托（代理）：利用冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件
