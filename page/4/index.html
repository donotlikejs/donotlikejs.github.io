<meta http-equiv="X-UA-Compatible" content="edge">

X-UA-Compatible 是针对IE8版本的一个特殊文件头标记，content=”edge” 通知IE用最高级别的可用模式显示内容

http-equiv 类似于http头部协议，用来回应给浏览器一些信息，以帮助浏览器正确的现实网页内容。
一些平常使用的http-equiv的名值对

//10s后刷新页面
<meta http-equiv="refresh" content="10">
//10s后跳转到bing
<meta http-equiv="refresh" content="10" url="http://bing.com">

git简明操作

2016-12-21
other

基本设置

​ - 查看配置信息

git config --list

​ - 设置 姓名和邮箱地址 (存放于 “~/.gitconfig”)

    git config —global user.name “yourname”
    git config —global user.email “yourEmail”

    git config —global color.ui auto

​ - 创建 SSH Key

    ssh-keygen -t -C “lianggunzai@gmail.com”

​ - 查看 id_rsa.pub（公开密钥）( id_rsa 私有密钥 )

    cat ~/.ssh/id_rsa.pub

​ - 和github进行认证

    ssh -T git@github.com

从远程仓库获取

    #获取远程仓库
    git clone git@github.com:Jianru-Lin/lambda-view.git

    #查看当前目录还没有被git管理和被git管理且被修改但还没有提交的文件
    git status    

    #查看当前分支的相关信息     (-a参数可以同时显示本地仓库和远程仓库的分支信息)
    git branch -a 

    #文件加入暂存区
    git  add   将文件加入暂存区

    #提交（加入git树）
    git commit -m “解释短语” 

    #更新
    git push

本地建立项目并且推送到远程仓库

    #首先先建立一个文件夹并进入
    mkdir gittest
    cd gittest

    #初始化仓库（目的是为了产生“.git”目录，里面存放管理当前目录内容的数据，数据也称为附属该目录的工作树）
    git init

    #查看仓库状态
    git status

    #提交到暂存区
    git add filename

    #更新（提交到工作树）
    git commit -m “解释说明短语”

    #添加远程仓库
        git remote add 仓库路径

    #推送至远程仓库
        git push -u origin master         推送至master分支
        git push -u origin 其他分支    推送到其他分支
        (-u参数可以在推送到同时，将origin仓库的master分支设置为本地仓库当前分支的upstream （上游）)

其他命令

分支 branch

    #查看分支一览表和显示当前分支
    git branch

    #已当前branch分支为基础创建新分支并且切换到新建的分支
    #等同于 git branch fenzhia ; git checkout fenzhia
    git checkout -b fenzhia  

   #切换分支
    git checkout branch-name
    #切换回上一个分支
    git checkout - 

    #合并分支
        #1.先切换到master主分支
    git checkout master
        #2.再合并
        #参数 —no-ff 可以确保历史记录中明确记录下本次分支合并
    git merge —no-ff fenzhia

合并冲突处理

版本回溯和推进

查看

After doing this, you may fix the identity used for this commit with:
设置完成以后，你可以通过以下这条语句来修复用于此提交到身份

git commit --amend --reset-author

查看已提交到日志
git log 

参数 
    —pertty=short 只显示第一行参数
    加文件名或目录名 只会显示相关的日志
    -p 显示文件的前后差别

查看更改前后的差别
git diff

            参数 HEAD 查看与最新提交的差别 （好习惯：git commit 前先 git diff HEAD 查看区别）

以图表形式查看分支
git log –graph

回溯历史分支（跑回之前修改的某个地方）

git reset —hard 哈希值 （哈希值通过git log 能找到 复制粘贴即可）

git reflog 查看当前仓库的所有日志
