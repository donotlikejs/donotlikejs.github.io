事件event

2017-03-03
javascript

事件：用户或者浏览器自身执行的某种动作。

事件流 页面中接收事件的顺序。

事件捕获：从不具体的父元素逐步传到具体的元素（Netscape定义的事件流）

事件冒泡：具体的元素逐步往上传播到不具体的父元素（IE定义的事件流）

DOM2级定义的DOM事件流:分为事件捕获阶段，处于目标阶段和事件冒泡阶段。（但浏览器都支持捕获和冒泡阶段对目标对象进行操作）

事件处理程序（事件监听器）：响应某个事件的函数，事件处理程序的名字以“on”开头。

事件委托（代理）：利用冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件.

事件绑定&事件监听&事件委托

  • 事件绑定（DOM0级定义）

    • 将一个函数赋值给一个事件处理程序属性
      btn.onclick=function(){
  }
    • HTML事件处理程序 ：写在html标签里
      <a onclick="changeColor()">change color</>

  • 事件监听（DOM2级定义的方法）

         /*
      *1参数为要处理的事件名
      *2参数事件处理程序的函数
      *3参数为布尔值，true表示捕获阶段调用2，false表示冒泡阶段调用2fn在当前作用于运行
      */
      addEventListener("",fn,boolean)
      /*
       *参数同上，只是调用变成删除
       */
      removeEventListener()
  • DOM3级:事件委托

IE事件处理程序：

// fn会在全局作用域中运行
attachEvent()
detachEvent()

事件对象（event）

触发DOM事件时自动产生事件对象event，该对象包含所有与事件有关的信息。

stopPropagation() 停止派发事件
preventDefault()   取消默认行为
target                返回最初触发事件的元素
type                返回事件类型

webpack基础应用

2016-12-22
工具

官网：https://webpack.js.org/

webpack是一款模块加载和打包的工具

1.全局安装

 npm install -g webpack

进入你的项目文件夹（比如叫yourproject）和 初始化

cd yourproject
npm init

初始化用来生成package.json包

{
  "name": "webpacktest",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "webpack": "^1.14.0"
  }
}

安装依赖

npm install webpack --save-dev

生成mode_modules文件夹以及在package.json的devDependencies中自动添加

"devDependencies": {
    "webpack": "^1.14.0"
  }

在每个项目根目录下必须配置一个webpack.config.js文件，此文件用来写配置项，用来告诉webpack做什么。
webpack.config.js里一般写法

//var webpack=require('webpack');
//var commonPlugin = new webpack.optimize.CommonsChunkPlugin('comon.js');


module.explorts={
  //plugins:[commonPlugin],
  //entry里为设置处理的文件路径入口
  entry:{
    index:'./src/js/page/index.js'
    },
  //output里为处理后存放的文件夹以及新的文件名
  output:{
    path:'dist/js/page',
    //publiPath:'',
    filename:'[name].js'
    },
  // module这里定义了处理指定后缀文件的插件，插件需要安装在项目内
  // npm install 插件名 -dev--save
  module:{
    loaders:[
      //处理css和scss文件
        {test:/\.css$/,
         loader:'style-loader!css-loader!sass-loader'
        },
      //处理es6文件
        {
          test:/\.js$/,
          //排除不需要解析道文件夹
          exclude:(),
          loader:'label-loader',
          //
          qurey:{
            presets:['es2015']
          }
        }

      ]
    },
  //
  resolve:{
    // 设置查找module的根目录，可忽略
    root:'',
    // resolve.extensions设置扩展文件后缀名， require()可以省略文件名
    extensions:['','js','css'],
    //设置模块别名
    alias:{

      }
    },
  //插件
  plugins:{

  }
}

打印shell的版本

环境变量PATH的查看，添加

  • 在自主安装的文件经常会用到shell来启动执行，有时候需要自行把程序添加到$PATH之中,当输入可执行程序后，会搜索这个目录列表。
  • mac中有三个级别对环境变量，分别是 系统变量 全局级变量和用户级变量，我们一般只修改 用户级别的环境变量 。
  • 保存环境变量的文件名为 .bash_profile ,位于 home根目录 下。
  • 查看PATH
echo $PATH
  • 添加path
export $PATH=路径
  • 立即生效
source ~/.bash_profile
