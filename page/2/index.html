事件event

2017-03-03
javascript

事件：用户或者浏览器自身执行的某种动作。

事件流 页面中接收事件的顺序。

事件捕获：从不具体的父元素逐步传到具体的元素（Netscape定义的事件流）

事件冒泡：具体的元素逐步往上传播到不具体的父元素（IE定义的事件流）

DOM2级定义的DOM事件流:分为事件捕获阶段，处于目标阶段和事件冒泡阶段。（但浏览器都支持捕获和冒泡阶段对目标对象进行操作）

事件处理程序（事件监听器）：响应某个事件的函数，事件处理程序的名字以“on”开头。

事件委托（代理）：利用冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件.

事件绑定&事件监听&事件委托

  • 事件绑定（DOM0级定义）

    • 将一个函数赋值给一个事件处理程序属性
      btn.onclick=function(){
  }
    • HTML事件处理程序 ：写在html标签里
      <a onclick="changeColor()">change color</>

  • 事件监听（DOM2级定义的方法）

         /*
      *1参数为要处理的事件名
      *2参数事件处理程序的函数
      *3参数为布尔值，true表示捕获阶段调用2，false表示冒泡阶段调用2fn在当前作用于运行
      */
      addEventListener("",fn,boolean)
      /*
       *参数同上，只是调用变成删除
       */
      removeEventListener()
  • DOM3级:事件委托

IE事件处理程序：

// fn会在全局作用域中运行

detachEvent()

修饰符

/*

    i    不区分大小
    g    全局匹配
    m    多行匹配

*/

元字符 . \b \w\d\s

/*
    .          匹配任意字符
    \b         匹配单词边界
    \w        匹配数字，字母，下划线
    \d        匹配数字
    \s        匹配空格
    [abc]    匹配方括号里任意一个字符
    ^        匹配首部
    $        匹配尾部

    --取反--
    \B \W \D \S [^abc]

*/

量词 * + ? {}

/*

    *        匹配任意个
    +        匹配一到多个
    ?        匹配 0 到 1 个
    {n}        匹配 n 个
    {n,}    匹配至少 n 个
    {n,m}    匹配 n 到 m 个

*/

分枝条件 |

/*

a|b         匹配 a 或者 b 

*/

分组 ()

/*

    (exp)            匹配且分组
    (?<name>exp)    匹配且以name名分组
    (?:exp)            只匹配不捕获不分组
    (?=exp)            匹配exp前面位置
    (?<=exp)        匹配exp后面位置
    (?!exp)            匹配后面不跟着exp的位置
    (?<!exp)        匹配前面不跟着exp的位置
    (?#comment)        注释

 */

懒惰限定符

/*

    *?
    +?
    ??
    {n,m}?
    {n,}?

*/

转义字符 \

/*

    需要转义的字符   * . ? + $ ^ [ ] ( ) { } / \ |
