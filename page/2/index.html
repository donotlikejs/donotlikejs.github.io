webpack基础应用

发表于 2016-12-22 | 分类于 工具

官网：https://webpack.js.org/

webpack是一款模块加载和打包的工具

1.全局安装

 npm install -g webpack

进入你的项目文件夹（比如叫yourproject）和 初始化

cd yourproject
npm init

初始化用来生成package.json包

{
  "name": "webpacktest",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "webpack": "^1.14.0"
  }
}

安装依赖

npm install webpack --save-dev

生成mode_modules文件夹以及在package.json的devDependencies中自动添加

"devDependencies": {
    "webpack": "^1.14.0"
  }

在每个项目根目录下必须配置一个webpack.config.js文件，此文件用来写配置项，用来告诉webpack做什么。
webpack.config.js里一般写法

//var webpack=require('webpack');
//var commonPlugin = new webpack.optimize.CommonsChunkPlugin('comon.js');


module.explorts={
  //plugins:[commonPlugin],
  //entry里为设置处理的文件路径入口
  entry:{
    index:'./src/js/page/index.js'
    },
  //output里为处理后存放的文件夹以及新的文件名
  output:{
    path:'dist/js/page',
    //publiPath:'',
    filename:'[name].js'
    },
  // module这里定义了处理指定后缀文件的插件，插件需要安装在项目内
  // npm install 插件名 -dev--save
  module:{
    loaders:[
      //处理css和scss文件
        {test:/\.css$/,
         loader:'style-loader!css-loader!sass-loader'
        },
      //处理es6文件
        {
          test:/\.js$/,
          //排除不需要解析道文件夹
          exclude:(),
          loader:'label-loader',
          //
          qurey:{
            presets:['es2015']
          }
        }

      ]
    },
  //
  resolve:{
    // 设置查找module的根目录，可忽略
    root:'',
    // resolve.extensions设置扩展文件后缀名， require()可以省略文件名
    extensions:['','js','css'],
    //设置模块别名
    alias:{

      }
    },
  //插件
  plugins:{

  }
}

新手小问题收集

发表于 2016-12-22 | 分类于 html 
<meta http-equiv="X-UA-Compatible" content="edge">

X-UA-Compatible 是针对IE8版本的一个特殊文件头标记，content=”edge” 通知IE用最高级别的可用模式显示内容

http-equiv 类似于http头部协议，用来回应给浏览器一些信息，以帮助浏览器正确的现实网页内容。
一些平常使用的http-equiv的名值对

//10s后刷新页面
<meta http-equiv="refresh" content="10">
//10s后跳转到bing
<meta http-equiv="refresh" content="10" url="http://bing.com">

git和github

发表于 2016-12-21 | 分类于 基础

基础

​ + 查看配置信息

git config --list

​ + 设置 姓名和邮箱地址 存放于 “~/.gitconfig”

    git config —global user.name “yourname”
    git config —global user.email “yourEmail”

    git config —global color.ui auto

​ + 创建 SSH Key

    ssh-keygen -t -C “lianggunzai@gmail.com”

​ + 查看 id_rsa.pub（公开密钥）( id_rsa 私有密钥 )

    cat ~/.ssh/id_rsa.pub

​ + 和github进行认证

    ssh -T git@github.com

​ + 从远程仓库获取

    git clone git@github.com:Jianru-Lin/lambda-view.git

git status    
查看当前目录还没有被git管理和被git管理且被修改但还没有提交的文件
查看当前分支的相关信息     -a     参数可以同时显示本地仓库和远程仓库的分支信息
git branch -a 
git  add    将文件加入暂存区
git commit -m “解释短语”   提交

git push    更新

本地建立项目并且推送到远程仓库

首先先建立一个文件夹并进入
    mkdir gittest
    cd gittest

初始化仓库（目的是为了产生“.git”目录，里面存放管理当前目录内容的数据，数据也称为附属该目录的工作树）
    git init

查看仓库状态
    git status

提交到暂存区
    git add filename

更新（提交到工作树）
    git commit -m “这里是解释说明短语”


After doing this, you may fix the identity used for this commit with:
设置完成以后，你可以通过以下这条语句来修复用于此提交到身份

    git commit --amend --reset-author
查看已提交到日志
    git log 

    参数 
        —pertty=short 只显示第一行参数
        加文件名或目录名 只会显示相关的日志
        -p 显示文件的前后差别

查看更改前后的差别
    git diff

                参数 HEAD 查看与最新提交的差别 （好习惯：git commit 前先 git diff HEAD 查看区别）


分支操作（重点难点）

显示分支一览表
    git branch

已当前branch分支为基础创建新分支并且切换到新建的分支
git checkout -b fenzhia  等同于 git branch fenzhia ; git checkout fenzhia（可以用 git branch 查看当前工作分支）

   切换分支
git checkout 分支名

git checkout - 切换回上一个分支

合并分支
先切换到master主分支
git checkout master
再
git merge —no-ff fenzhia （参数—no-ff 可以确保历史记录中明确记录下本次分支合并）

随后编辑器会启动（vi编辑器，直接退出或者，编辑再退出都ok）
然后结束

以图表形式查看分支
git log --graph

回溯历史分支（跑回之前修改的某个地方）

git reset —hard 哈希值 （哈希值通过git log 能找到 复制粘贴即可）


git reflog  查看当前仓库的所有日志

冲突解决
如果git merge --no—ff 分支 有冲突

打开编辑器 把内容修改好后 执行 git add 文件名；git commit -m “解释短语”
